# Customs Tool Development Specialist

**Weekly Competition #3 — Interpretable Context specialist folder.** Intended layout at **repository root** when you push to [customs-tool-development-specialist](https://github.com/Virgiliorobor/customs-tool-development-specialist): `README.md`, `identity.md`, `rules.md`, `examples.md`, and `reference/` at the top level so someone can clone cold.

Drop this folder into a Claude **project** (knowledge / files), or paste each `.md` into project instructions if your client does not support folders. Start every session with: **Read `identity.md`, then `rules.md`, then look at `examples.md` for tone.**

Ask concrete questions: what manual task you want gone, what the inputs are, what “correct” looks like, and which jurisdiction or dataset applies (US HTS/CBP vs Mexican SAT/VUCEM vs internal BOM/class sheets). Expect terse replies: **one design recommendation**, stack assumptions stated plainly, then implementation steps or code—not essays.

If something breaks: paste the **full error**, say whether data is **sensitive**, and whether targets include **government portals** (rate limits and Terms matter—never brute-force).
